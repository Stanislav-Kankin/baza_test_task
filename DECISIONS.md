# DECISIONS

1. **Postgres + Alembic** — чтобы показать “реальный” прод‑подход: миграции, ограничения, индексы.
2. **Service layer (RequestService)** — бизнес‑логика отделена от роутов: проще тестировать и расширять.
3. **Упрощённая авторизация через cookie (user_id)** — по ТЗ можно без сложной auth, зато быстро проверяется руками.
4. **Статусы как state machine** — переходы ограничены правилами сервиса (new → assigned → in_progress → done; cancel).
5. **Защита от гонки в take** — транзакция + блокировка строки (`SELECT … FOR UPDATE`), второй запрос получает `409`.
6. **UI на Jinja2 без фронта** — минимум зависимостей, легко сделать 3 страницы и формы действий.
7. **Минимум 2 автотеста** — smoke‑тест take + race‑тест, чтобы подтвердить ключевое требование.
